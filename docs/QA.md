# 月読 QA手順

この手順は、静的フロントエンドとしての公開前確認を想定しています。

## 事前準備

```bash
npm install
npm run dev -- --port 3000
```

Viteが別ポートへフォールバックした場合は、以降のURLのポートだけ読み替えます。

## コマンド確認

```bash
npm run check
git diff --check
```

期待結果:

- TypeScript型チェックが成功する。
- Vitestが全件成功する。
- Vite production buildが成功する。
- `git diff --check` が空で終了する。

## 通常表示

URL: `/`

確認項目:

- 「月読」「墨暦からくり時計」が表示される。
- 稽古パネルがDOMにも画面にも出ない。
- 東京が初期観測地になっている。
- 日付、和暦風ラベル、曜日、二十四節気、七十二候、月齢が読める。
- Consoleに関連する `error` / `warn` がない。

## 稽古パネル

URL: `/?keiko=1`

確認項目:

- 稽古パネルが表示される。
- 「現在へ戻す」「再生/停止」「±1秒」「±1分」「±1時間」「±1日」が操作できる。
- 分歯車、毎時小窓、日付落款、節気滲み、候滲み、月齢窓の各ボタンで表示時刻または演出パルスが変わる。

固定時刻/低モーション確認URL:

`/?keiko=1&at=2026-07-05T06:30:00%2B09:00&speed=60&play=0&motion=reduce`

期待結果:

- パタパタ時計が `06:30:00` を示す。
- `data-reduced-motion="true"` になり、重い演出が抑制される。
- Consoleに関連する `error` / `warn` がない。

## 都市切替

手順:

1. 通常表示または稽古表示で観測地を東京から金沢へ切り替える。
2. 観測地ラベルとステータスが金沢へ変わることを確認する。
3. 日の出、日の入り、太陽高度、月高度が東京時と異なる値へ更新されることを確認する。

## レスポンシブ確認

1280x720:

- 横スクロールが出ない。
- 通常表示では主要要素が一画面に収まる。
- 稽古表示では作品本体と稽古パネルが重ならず、縦スクロールで読める。

390x844:

- 横はみ出しがない。
- 日付、時計、パタパタ、暦、日月情報、稽古パネルの順に読める。
- 長い日本語ラベルがボタンやパネルから不自然にはみ出さない。
